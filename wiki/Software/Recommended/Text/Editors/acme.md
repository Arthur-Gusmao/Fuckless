## Acme

A text editor, user interface, shell, and programming environment from Plan 9.

Acme is not just an editor.

It is an environment where editing, programming, and interacting with the operating system become a single workflow.

Acme demonstrates that powerful software does not require unnecessary complexity.

Many features considered "advanced" in modern editors are achieved through a much simpler design.

### In Acme everything is text

In Acme, filenames, commands, compiler output, URLs, and error messages are all editable text.

There is no distinction between "editor" and "terminal".

If you can see text, you can interact with it.

### Mouse as a first-class interface

Unlike Vim and Emacs, Acme was designed around efficient mouse interaction.

The mouse is not an afterthought.

Chording and multiple buttons allow navigation, execution, and editing with very little friction.

### The shell is part of the editor

There is no artificial separation between editing and executing commands.

Running programs, compiling code, inspecting output, and navigating source files are all part of the same environment.

# Inclusion Criteria

The goal of this wiki is to curate software that values simplicity, transparency, and user control. Every listed project is evaluated using the following criteria.

## Required

Projects should satisfy most or all of these requirements.

* Clear documentation
* Free and Open Source Software
* Hackable codebase
* High code quality
* No bloat
* No bullshit
* No unnecessary complexity
* Portable
* Simple and understandable design
* Stable
* Unix philosophy

## Principles

Software listed here should:

* Do one thing well
* Be understandable
* Be maintainable
* Be composable
* Respect the user's freedom
* Respect system resources
* Avoid unnecessary abstractions

---

## Notes

This wiki is opinionated.

Minimalism does **not** necessarily mean fewer lines of code or fewer features. Instead, we value software that solves its problem with the least reasonable complexity while remaining reliable, maintainable, and respectful of the user.

If a piece of software needs an essay to justify its design, it probably does not belong here.

Simplicity is not a limitation. It is a feature.
